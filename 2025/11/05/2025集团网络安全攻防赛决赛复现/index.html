非关系型数据库

这是一道RDG的题目，登上去发现环境是java ，用的spring boot。

这题看了老师的WP，修复方式：

修改NoSQLDataBase类将函数写法变成预加载

  1. 参数类型限制：Controller 中 username 和 password 明确声明为 String 类型，Spring MVC 会自动将用户输入解析为字符串（即使攻击者传入 {"$ne":""}，也会被解析为字符串 "{"$ne":""}"，而非 DBObject）；
  2. Driver 自动转义：MongoDB Java Driver 的 BasicDBObject.put() 方法，当值为字符串时，会自动将其视为「字面量字符串」，不会解析为查询操作符（如 $ne、$or）。

重点是后面class文件的修改和编译，尝试过程如下：

最开始是用IDEA来打开jar文件(IDEA打开，file->open，选中jar包所在的那个文件夹)，导入文件夹之后，在文件夹下找到jar文件，右键找到选项Add as Library 。就可以把class文件都导入进去了。如下：

但是发现文件都只是read-only权限，于是用了一种很笨的方法，先是在服务器上面找到jar文件，然后jar xf mongo.jar 解压缩成三个文件夹。本地新建了一个NoSQLDataBase.java文件，代码改好，传到服务器上面对应的文件夹下面，用服务器自身环境进行编译，参考了柏总的命令：javac -cp ".:BOOT-INF/classes:BOOT-INF/lib/*" BOOT-INF/classes/com/example/ezjava/model/NoSQLDatabase.java 。生成的新的class之后，重新再到服务器上面打包成jar。jar cf mongo.jar BOOT-INF META-INF org 发现这样生成的jar包是有问题的。。。

其实就可以直接用IDEA自带的jaredit插件来直接修改jar包：

这里题目服务器的环境是jdk17，然后IDEA 2024.1版 自带的SDK Default 也是jdk17。所以这里直接编译没问题，save保存之后就行了。直接修改jar包。

轻量级数据库

看了老师的wp，修复点在：

通过反编译工具打开，发现存在数据连接测试接口

在依赖中发现了h2和mysql的组件，可以利用h2组件达到命令执行， H2 数据库 JDBC URL 注入漏洞

INIT=是 H2 JDBC URL 中用于 “初始化执行脚本” 的关键参数，攻击者常通过INIT=RUNSCRIPT或INIT=CREATE ALIAS注入恶意 Java 代码 / 脚本（如 CVE-2021-42392 的 JNDI 注入、CVE-2018-10054 的代码执行），过滤INIT=可直接切断这类攻击的核心触发路径；

漏洞修复

在连接测试的地方添加字符串过滤，将 INIT= 拦截掉即可

然后将这个类重新编译后，替换存在漏洞的类，重新打包上传即可修复

和上题非关系型数据库修改方式一致，直接去修改的jar包，不过编译的时候需要注意，题目服务器的环境是jdk1.8，所以我们本地用jaredit编辑的时候也需要jdk1.8
